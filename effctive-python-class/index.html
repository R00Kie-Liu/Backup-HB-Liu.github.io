类与继承

1. 用辅助类维护程序状态，而不是字典/元组

Python 教学中最经典的实例就是编写一个为各学生添加成绩的代码，当编辑内容只有学生姓名和对应成绩时，使用一个字典就能够轻易维护，当要求添加学生各科成绩时，我们也能通过构建字典的字典 来维护；进一步，如果还需要记录每次成绩的权重，那我们只能通过修改内部的字典，把 key(科目) 对应的键值修改为一系列元组：(score, weight) 来实现。

此时，外层字典的结构已经非常复杂，出现了字典套字典套元组的情况。当需要对学生成绩进行一些运算操作，如求均分时，会出现for循环嵌套for循环的尴尬现象。

当嵌套多与一层时，就应该避免使用字典的字典/元组这种做法，这会使得代码很难读懂，维护起来也麻烦。

所以，用来保存程序状态的数据结构一旦过于复杂，就应该将其拆解为类，以便提供更为明确的接口，并更好地封装数据。

  • 把嵌套结构重构为类：

嵌套结构使用辅助类重构的思想是由下至上，从最底层开始向上一层层构建，在该问题中，就是从每次考试的成绩开始。这么简单的信息，我们可以不用专门自己写一个类，由于分数和权重都不再变化，我们用元组就足够了。但是问题在于，普通的元组只是按照位置来排布其中的各项数值，如果我们除 score, weight 之外还要加入一些信息，如老师的评语，那就需要重新修改原来使用二元组的那些代码，随着元组长度逐步扩张，代码也趋于复杂。元组里的元素一旦超过两项，就应该考虑用其它办法来实现了。

collection 模块中的 namedtuple(具名元组) 非常适合这种需求：

import collection
Grade = collection.namedtuple('Grade', ('score', 'weight'))

构建具名元组时可以使用位置参数或者关键字参数，它除了可以像普通元组一样索引、赋值之外，最重要的就是可以通过属性访问各字段, 所以当需求发生变化时，就很容易从namedtuple迁移到自己定义的类

g = Grade(90, weight=0.9) # instantiate with positional or keyword arguments
s, w = g  # unpacke like a regular tuple
print(g.score * g.weight)  # accessible by name
>>>
81.0

接下来写表示科目的类，该类包含一系列考试成绩：

class Subject(object):
    def __init__(self):
        self._grades = []

    def report_grade(self, score, weight):
        return self._grades.append(Grade(score,weight))

    def average_grade(self):
        total, total_weight = 0, 0
        for grade in self._grades:
            total += grade.score * grade*weight
            total_weight += grade.weight
        return total / total_weight

接下来写表示学生的类：

class Student(object):
    def __init__(self):
        self._subjects = {}

    def subject(self, name):
        if name not in self._subjects:
            self._subject[name] = Subject()
        return self.subject[name]

    def average_grade(self):
        total, count = 0, 0
        for subject in self._subjects.values():
            total += subject.average_grade()
            count += 1
        return total / count

最后编写包含所有学生考试成绩的容器：

class Gradebook(object):
    def __init__(self):
        self._students = {}

    def student(self, name):
        if name not in self._students:
            self.students[name] = Student()
        return self._students[name]

这种类的代码量虽然大一些，但是这种程序理解起来要容易的多，使用这种类的范例代码写起来也比原来更清晰、更易拓展：

book = Gradebook()
Lihua = book.student('Lihua')
math = Lihua.subject('Math')
math.report_grade(90, 0.3)
# ...
print(Lihua.average_grade())
>>>
90
